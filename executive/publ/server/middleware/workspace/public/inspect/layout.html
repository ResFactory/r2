Client-side RF Layout

rfUniversalLayout (exposed as clientLayout in the target build site / publication. It is often referred to colloquially as "client cargo" container and provides a unified browser-side global Javascript object that holds server-side data useful for the page when it's served to a user agent ("UA" or "client"). Within the Resource Factory Explorer (RFE) the inspectable rfUniversalLayout is referred to as inspectableRFUL and is available as part of activatePageFx.done.watch(({{ result: { inspectableRFUL } }}) => ...).

Since RF strives to generate all HTML on the server side almost all page layouts are finished at build time so the user agent client does not have access to context information such as server side routes or page origination context. The rfUniversalLayout instance available to each page as a global window.clientLayout variable which gives you get server-side layout data on the client, giving you the ability to perform UA or client side content updates, make server side API calls, etc.
